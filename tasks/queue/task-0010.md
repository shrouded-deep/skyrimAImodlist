## Description

Post-task-0004 MO2 inspection revealed 15 active plugins still have
`BSHeartland.esm` / `BSAssets.esm` as declared masters. These cause red
missing-master errors — the game will not load. Our task-0004 audit only
scanned WRLD records; Bruma integration ran deeper into NPC_, LVLI, ARMO,
ECZN, and other record types across core Nolvus Awakening patches.

**Affected active plugins:**
- COINMerchantExchange.esp
- Leveled List Patch.esp
- NewZombies - Cyrodiil Patch.esp
- Nolvus Awakening Armors Balance Patch.esp
- Nolvus Awakening Combat Scaling Overhaul - Encounter Zone.esp
- Nolvus Awakening Combat Scaling Overhaul - Enemy Level Mult.esp
- Nolvus Awakening DLC Patch.esp
- Nolvus Awakening Economy Patch.esp
- Nolvus Awakening Leveled Armors Patch.esp
- Nolvus Awakening Model Patch.esp
- Nolvus Awakening NPC Patch.esp
- Nolvus Awakening Seasonal Tree Mashup - Pine Forest.esp
- Nolvus Awakening Survival Patch.esp
- Nolvus Awakening Water Patch.esp
- Nolvus Awakening Zombies Integration.esp
- Synthesis.esp (already handled by task-0009)

**Two-phase fix:**

**Phase 1 (emergency — agent):** Disable all 15 affected plugins in Successor
`plugins.txt` to unblock game launch. The Nolvus Awakening patches are
important but non-critical for a test launch — the list is still functional
without them temporarily.

**Phase 2 (surgical — human xEdit):** For each plugin, open in xEdit, remove
Bruma-specific records, clean masters, save. Re-enable cleaned plugins.
Plugins that are purely Bruma-scoped (NewZombies Cyrodiil, COINMerchantExchange)
can remain disabled permanently.

## Phase 1 steps — emergency disable (agent executes immediately)

1. Assert MO2 is closed.
2. In Successor `plugins.txt`, remove (or comment out) all 15 affected plugin
   entries (excluding Synthesis.esp which is already removed).
3. Verify no remaining active plugin lists BSHeartland.esm or BSAssets.esm
   as a master.
4. Report new active plugin count.

## Phase 2 steps — xEdit surgery (human, after Phase 1)

For each plugin in the surgical list:

1. Temporarily re-enable `Beyond Skyrim - Assets` and `Beyond Skyrim - Bruma`
   mod folders in Successor so xEdit can load the plugins without warnings.
2. Open xEdit on Successor profile. Load the affected plugin + its masters.
3. In the left pane, right-click the plugin → "Apply Filter" → filter by
   "Record from BSHeartland.esm" or use the master filter to show only
   records injected from / overriding BSHeartland.esm records.
4. Select all filtered records → Remove.
5. Right-click plugin → "Clean Masters" to drop BSHeartland.esm and
   BSAssets.esm from the master list now that no records reference them.
6. Save. Repeat for each plugin.
7. Re-disable the Beyond Skyrim mod folders.
8. Re-enable all surgically cleaned plugins in Successor plugins.txt.

**Permanent disables** (Bruma-only content, no surgical treatment needed):
- NewZombies - Cyrodiil Patch.esp
- COINMerchantExchange.esp (Bruma merchant integration only)
