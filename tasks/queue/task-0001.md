## Description

Nolvus Awakening V6 ships with ENB as its rendering framework. The user
prefers Community Shaders (CS) but ENB may be too deeply integrated to
remove without significant rework. This task is a **research-only audit**
— no changes to the mod list. Output a findings doc that gives enough
information for a human go/no-go decision on the transition.

CS and ENB cannot run simultaneously. The question is not whether to
prefer one over the other aesthetically, but how much work a transition
would actually entail and whether any Nolvus components would be
irrecoverably broken by the removal of ENB.

## Research questions

### 1. ENB binary and preset
- Which ENB preset does Nolvus ship? Who authors it?
- Is the preset actively maintained for the current Nolvus version?
- Does the preset have documented CS equivalents or migration guides?

### 2. Weather and lighting stack
- Which weather mod does Nolvus use (Obsidian, NAT, Cathedral, etc.)?
- Which interior lighting overhaul (Lux, ELE, ELFX, etc.)?
- Are either tuned specifically for ENB (e.g. imagespace multipliers,
  fake interior lighting values that assume ENB postprocessing)?
- Does CS have a compatible weather/lighting setup for the same mods?

### 3. ENB light meshes
- Does Nolvus include any mods that ship ENB-specific particle light
  meshes (`.nif` files with ENB light nodes)? Common sources: ELFX,
  Inferno, ELE, custom fire/torch mesh packs.
- If so, are CS-compatible replacers available (e.g. via CS Light Limit
  Fix mesh packs, or Particle Patch for ENB → CS equivalents)?

### 4. ENB-specific shader mods
- List any mods in the Nolvus load order that are explicitly ENB-only
  features: ENB lens flare, ENB complex parallax, ENB particle lights,
  ENB rain occlusion configs, etc.
- For each, note whether CS 1.7.x has a native equivalent.

### 5. CS 1.7.x hard-block check
- Cross-reference the Nolvus mod list against the CS 1.7.x hard-block
  list (Light Limit Fix standalone, EVLaS, Terrain Shadows standalone,
  Water Effects standalone, etc. — see `E:\Skyrim AI Modlist\lessons-learned.md`).
- Note any that Nolvus currently ships as standalone mods.

### 6. Overall complexity estimate
- Summarise the transition as: **straightforward** (swap binary, pick CS
  preset, done), **moderate** (weather/lighting re-tune needed, mesh
  swaps available), or **major** (deep integration, significant content
  loss or rework required).

## Output

Write findings to `modlist/enb-cs-audit-2026-07-09.md`. Structure:
- ENB preset identity
- Weather/lighting stack summary
- ENB light mesh exposure (list mods, note CS replacer availability)
- ENB-only shader features (list, note CS parity)
- CS hard-block conflicts in current list
- Complexity verdict with reasoning

Do not make any changes to the mod list or MO2 profile. Research only.
