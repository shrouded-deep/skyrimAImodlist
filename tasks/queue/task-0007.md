## Description

The Nolvus Synthesis Patch mod folder contains only a baked `Synthesis.esp`
output (124 MB) and a `meta.ini` — no pipeline config is shipped. Synthesis
stores its pipeline in `%AppData%\Synthesis\`, which is machine-local and not
part of the Nolvus install.

Before the human runs Synthesis regen on the Successor profile, we need to
know exactly which patchers Nolvus V6 uses and in what order, so the regen
produces an equivalent ESP rather than an accidental subset or empty run.

## Steps

1. Check the Nolvus Awakening V6 official documentation and GitHub for the
   Synthesis patcher list:
   - https://www.nolvus.net (guide / appendix / install pages)
   - https://github.com/Styyx1/nolvus-awakening (or similar — search for
     the Nolvus GitHub org)
   - Nolvus Discord announcements if publicly accessible
2. Check whether `%AppData%\Synthesis\` on this machine already has a
   pipeline config (i.e. was Synthesis previously configured here):
   - Path: `C:\Users\steve\AppData\Roaming\Synthesis\`
   - Look for `PipelineSettings.pex`, pipeline JSON files, or a `settings`
     folder listing installed patchers
3. If the AppData config exists and lists patchers, record them — it may
   already reflect the Nolvus setup from the original install.
4. Compile findings into `modlist/synthesis-patcher-list.md`:
   - List of patchers (name, source repo/Nexus ID if known)
   - Load/run order
   - Any patcher-specific settings noted in Nolvus docs
   - Source of the information (doc URL, AppData config, etc.)
   - Any patchers that touched Bruma content and should be disabled/removed
     for the Successor regen

## Acceptance criteria

- [ ] Nolvus V6 documentation checked for Synthesis patcher list

- [ ] `modlist/synthesis-patcher-list.md` written with findings

- [ ] Result notes the information source and confidence level
