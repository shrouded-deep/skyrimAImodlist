## Description

Two distributor/animation-framework updates flagged in the version-check
report, bundled here since both affect wide-reaching frameworks but carry
very different risk levels — treat them as two distinct sub-tasks within
this one, not a single blind batch update.

**SPID**: 7.2.0.0.RC11 -> 7.2.0.0.RC19+. Routine RC bump, but SPID drives
`_DISTR.ini` files across many mods — test after updating.

**Open Animation Replacer**: 2.3.6.0 -> 3.1.6 (major version, released
June 2026). The version-check report explicitly flags this as high-risk
and recommends auditing every OAR/DAR-dependent mod before updating —
don't skip that step.

## Acceptance criteria

- [ ] **SPID:** update to latest RC, verify `_DISTR.ini` files across
      installed mods still parse/apply correctly after update
- [ ] **OAR:** identify every installed mod that depends on OAR or DAR
      (Dynamic Animation Replacer) conditions before touching anything
- [ ] For each OAR/DAR-dependent mod found, check whether it's confirmed
      compatible with OAR 3.x (check mod pages/changelogs, not just
      assumption) — OAR 3.x condition file format or behavior changes
      could silently break older condition files
- [ ] Write findings to a short section in `modlist/decisions.md` before
      deciding whether to proceed with the OAR update
- [ ] If everything checked is confirmed compatible, update OAR and
      verify load order still resolves cleanly
- [ ] If any dependent mod's compatibility is unclear or unconfirmed,
      stop and flag it — do not update OAR speculatively given its
      "high" risk rating in the version-check report
- [ ] Update `modlist/changelog.md` regardless of outcome (even a
      "deferred" decision is worth logging)
