## Description

task-0005 (full Cell conflict audit) found one conflict that looks
unintentional: `McmRecorder.esp` — a lightweight MCM-recording/debug
utility mod — wins cell `0BBCB2:Skyrim.esm` (editorID `WEMerchantChests`,
name "Warehouse Bookshelves") over `Update.esm`. The diff is not a routine
field tweak:

- The cell's `NavigationMeshes` goes from 1 item (vanilla) to 0 items in
  McmRecorder.esp's version — navmesh data is dropped for this cell.
- The cell's `Persistent` placed-reference list is swapped: vanilla
  references `00327E:Update.esm`; McmRecorder.esp's version instead places
  its own `MiscItem` reference (`000D65:McmRecorder.esp`, editorID
  `McmRecorder_MessageText` — an MCM-helper debug item).

This has the signature of an accidental/dirty edit picked up while the
mod author had this cell open in the Creation Kit for unrelated testing,
not an intentional design choice for a debug/MCM-recording mod. Full
detail (including the raw field diff) is in
`modlist/baseline-scan-2026-07-05.md`, "Full manual audit — Cell
(task-0005)" section, "Needs attention" subsection.

This task is research/decision work (check the mod's Nexus page /
description for any stated reason to touch this cell, check if this is a
known issue reported by other users, decide whether `WEMerchantChests` is
a player-visited cell where a missing navmesh would actually matter) — it
does not require houseCARL directly, which is why it's assigned to
Cursor. If the conclusion is that a corrective patch is needed, write a
new `task-00XX` (assigned to Claude Code, `requires_housecarl: true`)
specifying exactly what the patch plugin should restore (navmesh + the
original persistent reference) rather than doing that edit here.

**Note:** this was originally filed as task-0008, but that ID was taken
in parallel by a different task (mod version-check). Renumbered to
task-0009 — no content change.
