## Description

Project New Reign - Nemesis (60033) has had no updates since Dec 2021.
The user has decided to transition this list to its successor, **Pandora
Behaviour Engine+** (133232, v4.2.0-beta as of Mar 2026). This is a
list-wide change — Nemesis's output touches every animation-mod-driven
behavior file, and Anvil currently ships pre-generated Nemesis output —
so this needs real research before any execution task (task-0017) touches
the actual install.

## Acceptance criteria

- [ ] Research Pandora Behaviour Engine+'s compatibility model: does it
      consume the same animation mod inputs as Nemesis (FNIS/Nemesis-style
      behavior mods), or does it require mods to be Pandora-specific?
- [ ] Identify every installed mod in this list that depends on Nemesis
      output or is patched specifically for Nemesis (check mod
      descriptions/requirements, not just filenames)
- [ ] For each one, confirm whether it's known-compatible with Pandora,
      needs a Pandora-specific patch, or has no confirmed compatibility
      info yet
- [ ] Research the actual migration steps: does Pandora need to fully
      replace Nemesis (uninstall) or can they coexist during transition?
      What generated output needs to be cleared/regenerated?
- [ ] Check whether any other tools in the list (DynDOLOD, Synthesis
      patchers, etc.) have any dependency on Nemesis-specific output
      that would need to be re-run after migration
- [ ] Write a migration plan to `modlist/pandora-migration-plan.md`:
      what's compatible as-is, what needs patches, what's unconfirmed,
      and the recommended step order
- [ ] Add a summary entry to `modlist/decisions.md`
- [ ] Do not execute the migration in this task — write the plan and
      create a follow-up `requires_housecarl: true` task for Claude Code
      to execute once the plan is reviewed, per the standing rule on
      self-service task creation. Flag clearly if this should instead
      wait for explicit human sign-off given it's a list-wide change.
