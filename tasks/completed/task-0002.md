## Description

The user wants to slim down the content width of the list, starting with
section 4.3 (NEW QUESTS WORLDS & ADDONS) of the Nolvus modlist. Bruma
is a confirmed removal candidate; others may follow. This task audits
every mod in that section to answer two questions per mod:

1. Does it add a new worldspace (or significantly extend an existing one)?
2. How complex is removal — specifically, what Nolvus custom patches
   reference it, and what would need to be cleaned or removed alongside it?

This is **research only** — no changes to the Successor profile.

## Steps

### 1. Extract section 4.3 from modlist.txt

Read `E:\Nolvus\Nolvus\MODS\profiles\Nolvus Awakening\modlist.txt`.
Find the separator line for section 4.3 (likely named something like
`_separator:4.3 NEW QUESTS WORLDS & ADDONS` or similar) and extract all
enabled mod entries until the next separator. Record the full mod list.

### 2. For each mod — worldspace check

Use `housecarl_cross_plugin_query` or `housecarl_read_record` to check
whether each mod's ESP/ESM defines or edits WRLD records (worldspace).
Flag as:
- **New worldspace** — adds a WRLD record not in the base game
- **Worldspace edit** — edits an existing WRLD (e.g. Skyrim, Tamriel)
- **No worldspace** — quest/item/follower mod with no exterior world

### 3. For each mod — patch complexity

Check `plugins.txt` and the mods folder for any Nolvus custom patch
plugins that master the mod being assessed. Look for:
- Patches in the Nolvus Awakening patches section of the modlist
- Any LOOT/Nolvus-specific merge plugins that include it
- Lux, Lux Via, Lux Orbis patch slices for it
- SREX patch references (critical — see AGENTS.md city stack constraint)

Rate removal complexity as:
- **Clean** — disable mod + its plugin; no patch orphans
- **Moderate** — 1–3 patches need disabling or cleaning; straightforward
- **Complex** — deep patch web, custom Nolvus patches, or SREX involvement

### 4. Bruma specifically

Beyond Reach (Bruma) is a confirmed removal target. Give it full
treatment: list every plugin that masters it, assess whether any of those
patches touch other mods the user wants to keep, and note whether a
clean removal is possible without houseCARL record edits.
