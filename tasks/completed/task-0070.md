## MO2 separator rules — READ FIRST

In `modlist.txt`: **TOP of file = highest priority. A separator OWNS the mods listed
ABOVE it** (earlier in the file). To place mods under a separator in the MO2 UI, they
must appear ABOVE the separator line in modlist.txt.

**Separators are ALWAYS prefixed with `-` in modlist.txt.** This is correct and
intentional — MO2 does not try to "load" separator entries, so they remain disabled.
Do NOT change separator lines to `+`. Only regular mod entries use `+`.

For every separator line in modlist.txt, a matching empty folder **must** exist at
`D:\Skyrim\mods\<separator_name>_separator\`. Create it if missing or MO2 will silently
drop the separator on next save.

Example — correct placement (note separator stays `-`):
```
+JK's Bannered Mare          ← mod entry, enabled, owned by separator below
+JK's Blue Palace            ← mod entry, enabled, owned by separator below
-JKs Skyrim_separator        ← separator always disabled; owns everything above it
```

### Section order in modlist.txt (top = highest priority)

The three sections must appear in this order (top to bottom):

```
[City Stack mods]
-City Stack_separator
[Ryn's mods]
-Ryn's Mods_separator
[JK's interior mods]
-JKs Skyrim_separator
```
