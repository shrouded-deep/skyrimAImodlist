## Description

Live testing confirmed the Lux → MLO2 migration should be reverted. The
user has decided Lux's comprehensive patch ecosystem is worth keeping;
going forward, any mod lacking a Lux patch can be handled reactively via
a houseCARL patch rather than replacing the whole lighting philosophy.

A full pre-migration snapshot exists at
`modlist/exports/mlo2-pre-migration-2026-07-07/` — use it as the
restoration reference. The Lux CS Light JSONs were moved (not deleted)
to `_disabled_lux_mlo2_migration/` during the original migration and
should be restored from there rather than re-downloaded.

Note: task-0024's Synthesis fix (repointing `ANV_SynW4ENBPatcher`'s
broken local-path source to the public GitHub repo) is unrelated to Lux
and must NOT be reverted — that was an independent portability fix.
