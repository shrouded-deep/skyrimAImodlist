## Context

Task-0074 installed the settlement stack but five mods were not extracted
correctly — their archive contents were dumped directly into the mod folder
instead of being properly extracted. These appear greyed-out in MO2.

Additionally, all city and settlement mods ended up under the `Settlements`
separator, leaving `City Stack_separator` empty. It should be removed.

**Target profile:** `Lost Legacy - Fork` at `D:\Skyrim\profiles\Lost Legacy - Fork\`
**Always close MO2 before editing modlist.txt or plugins.txt.**

---

## Step 1 — Fix improperly extracted mods

The following mod folders contain raw archive contents instead of a proper
mod structure (no `Data\` folder or loose files at root — instead contains
the archive file itself or a subfolder matching the archive name):

- `D:\Skyrim\mods\The Great Town of Shor's Stone\`
- `D:\Skyrim\mods\RedBag's Rorikstead\`
- `D:\Skyrim\mods\The Great Village of Old Hroldan\`
- `D:\Skyrim\mods\Thuldor's Ivarstead - Tweaks and Fixes\`
- `D:\Skyrim\mods\Fortified Morthal Patch Collection\`

For each:
1. Delete the contents of the mod folder (not the folder itself)
2. Locate the correct archive in `E:\Modding\Skyrim Mods\Downloads\` —
   search by mod name if no Nexus ID is in the filename
3. Extract the archive so that the mod folder contains either a `Data\`
   subfolder or loose plugin/mesh/texture files at root level
4. Verify the mod is no longer greyed-out (has a valid structure)

**RedBag's Rorikstead** is a non-FOMOD mod — extract normally.
**Fortified Morthal Patch Collection** is a FOMOD — extract the main archive;
FOMOD option selection will happen during task-0075 (patch pass).
