## Description

task-0052 found that the VB install script did a **partial folder copy**: the
`Vokriinator Black` mod folder received its loose `.pex` scripts but NOT
`Vokriinator Black.esp` (the 529 KB core merge plugin), meta.ini, or source `.psc`
files. This passed all plugin-level checks except the missing-master warning because
the resolver only validates *plugin* files — it does not check meshes, textures, BSAs,
or scripts.

**Risk:** any other folder copied by `scripts/install-keizaal-vb-stack.ps1` or
`scripts/install-keizaal-city-stack.ps1` may be missing assets that won't surface until
in-game (purple textures, missing meshes, broken FX/scripts).

## Acceptance criteria

- [ ] Enumerate every mod folder the two install scripts copied from Nolvus
      (`E:\Nolvus\Nolvus\MODS\mods\`) into `E:\Skyrim\mods\` — derive from the script
      source lists, not guesswork
- [ ] For each, compare Keizaal copy vs Nolvus source: **file count** and **total byte
      size** per folder (recurse). Flag any folder where Keizaal < Nolvus
- [ ] For flagged folders, list the specific missing files
- [ ] Re-copy any incomplete folders from the Nolvus source (robocopy /E)
- [ ] Confirm `Vokriinator Black` folder is complete (should be — fixed in task-0052;
      re-verify file count matches Nolvus source = 20 items)
- [ ] Root-cause the script bug: why did the esp copy get skipped? Fix the script so
      future copies are atomic/verified (e.g. robocopy with exit-code check per folder)
- [ ] Record findings + any re-copies in Result; update decisions.md if material

## Result

**Status: COMPLETE — 2026-07-12. No other folders affected; root cause fixed.**

### Source correction

The VB script copies from **Nolvus** (`E:\Nolvus\Nolvus\MODS\mods`); the **city** script
copies from **Anvil** (`D:\Skyrim AI Modlist\Anvil\mods`) — not Nolvus. Integrity
comparison ran each folder against its true source.

### Integrity comparison (file count + total bytes, recursive)

Compared all **44 VB folders** (vs Nolvus) and **11 enabled city folders** (vs Anvil;
the 9 dropped Spaghetti folders were skipped — moot per task-0055).

**Result: every copied folder has dst ≥ src for both file count and total bytes.**
No incomplete folders. The `Vokriinator Black` folder (re-copied from Nolvus in
task-0052) now matches the source. Script: `scratchpad/folder-integrity.ps1`.

### Root cause + fix

The partial copy was **isolated to the `Vokriinator Black` folder**, the only one using
an extract-and-move-inner code path (VB script lines 106-108). All other folders used
`robocopy /E` (reliable full recursive copy) — confirmed complete.

The bug: after 7z extraction, the script moved `Get-ChildItem $tmp -Directory |
Select-Object -First 1` as the mod root. The VB archive had `Vokriinator Black.esp` at
root **plus** a `scripts` subfolder — so it grabbed `scripts` and moved only the `.pex`
files, dropping the `.esp`.

**Fixed** in `scripts/install-keizaal-vb-stack.ps1`: only descend into a wrapper dir if
`$tmp` holds exactly one item and it's a directory with no root-level esp/esm/esl/bsa;
otherwise treat `$tmp` itself as the mod root. Added a post-extract assertion that
`Vokriinator Black.esp` exists (throws if not).

### Verdict

Load order is safe for the VB new-game smoke test. Any CTD now is attributable to the
known Combat Styles issue (→ Crash Workaround 146503), not a missing asset.
