## Description

The Nolvus Synthesis Patch mod folder contains only a baked `Synthesis.esp`
output (124 MB) and a `meta.ini` — no pipeline config is shipped. Synthesis
stores its pipeline in `%AppData%\Synthesis\`, which is machine-local and not
part of the Nolvus install.

Before the human runs Synthesis regen on the Successor profile, we need to
know exactly which patchers Nolvus V6 uses and in what order, so the regen
produces an equivalent ESP rather than an accidental subset or empty run.

## Result

**Completed:** 2026-07-09 (Cursor)

### Findings

- **Full V6 pipeline not recoverable.** Nolvus Dashboard ships pre-baked `Synthesis Patch` v6.0.15 (~124 MB ESP only). No `PipelineSettings.json` in Nolvus install; `%AppData%\Synthesis\` absent on this machine.
- **Official docs:** [Ascension Synthesis guide](https://www.nolvus.net/guide/asc/output/synthesis) documents **Water Does Damage Patcher** + **Remove Landscape Vertex Color** + ccKRTSSE001QNWorld xEdit delete + ESLify. Awakening-specific Synthesis page **404**. Homepage notes Awakening has **no manual guide yet**.
- **False lead:** `E:\Modding\Skyrim Mods\tools\Synthesis\PipelineSettings.json` exists but targets **Anvil/Skyrim AE** with 4 unrelated patchers — not Nolvus.
- **Baked ESP header scan:** 55 master plugins including `BSHeartland.esm`, `BSAssets.esm`, `AI Overhaul.esp`, `Personalized Music v 6.0.esp`, Northern Roads, Water for ENB patches, etc. Patcher names inferred from masters are **hypothesis only** (documented in output file).
- **Bruma:** Baked ESP masters Bruma ESMs and (per task-0004) wins two CYR worldspaces. Successor regen must exclude Bruma from load order and verify new output has no BSHeartland masters.
