## Description

Live in-game testing found: clicking a chair/stool/bench triggers no
camera adjustment and no animation at all — not glitchy, not
misaligned, nothing happens. Mounting a horse shows the same complete
failure. TDM (headtracking, leaning, horse archery — also
Pandora-generated) works correctly, as does Community Shaders. This
narrows the problem to something specific in Pandora's output affecting
the furniture/mount interaction behavior category specifically, not a
wholesale regen failure.

Sitting and mounting share the same underlying behavior-graph mechanism
in Skyrim (furniture-state transition), so both failing together points
to one shared broken/missing piece rather than two unrelated bugs.

This is diagnosis-first. Do not attempt fixes until the actual cause is
identified — guessing wrong wastes a full cycle on the wrong system.

## Result

### Summary

Diagnosis completed **without a graph-level confirmed root cause**. ASCII
string / file-presence analysis shows Pandora's compiled output **does
contain** the expected furniture/mount event names at the same counts as
the Pandora template HKX and Nemesis SSE cache — so this is **not** a
simple "missing IdleFurnitureStart / HorseEnter strings" failure. The
`turn` WARN, XPMSSE double-patch, and bseaf/gmill patch failures are
**ruled out**. The most plausible remaining cause is a **transition wiring
defect inside compiled HKX** that string search cannot see — requiring
HKX→XML graph diff or in-game event tracing before any fix is applied.

**Confidence:** ruled-out items = high; primary hypothesis = **medium**
(pending graph diff / in-game diagnostic in task-0031).

---

### 1. `turn` WARN — ruled out as root cause (high confidence)

**Log line:** `WARN : Dispatcher > "Turning Animation Won't Affect UpperBody" >
defaultfemale~1hm_behavior > Replace > Text > #0426/modifiers > FAILED`

**What `#0426` is:** `PlayerBowModList` — a `hkbModifierList` in
`1hm_behavior` (one-handed bow attack modifier chain). Patch `#0426.txt`
inserts `#turn$0` (`BowAttack_ML`) into that list's modifiers array.

**Why it cannot cascade to furniture/mount:** Furniture uses `0_Master`
(`IdleFurnitureStart`, `IdleChair*`) → `mt_behavior` (`to_FurnitureState`,
`FurnitureState`). Mounting uses `0_Master` + `horsebehavior` (`HorseEnter`).
The failed replace targets **female 1hm bow modifiers only** — a graph file
not on the furniture/mount activation path. Patch still registers as applied
(`Pandora Mod 25` in `Engine.log`); Pandora reverted that one node per its
error-tolerance model.

---

### 2. XPMSSE double-patch — ruled out (high confidence)

- `ActiveMods.json` has **no `xpmsse` entry** among 28 patch codes.
- `Universal Behaviour Runtime - Auto Skeleton Patch` is enabled in MO2
  (`SkeletonAutoPatch.dll`).
- Pandora manual-run checklist item "do NOT enable Pandora XPMSSE patch" is
  satisfied.
- GitHub #495 ("can't sit after XPMSE patch") was user error (patch loaded
  last in MO2) — not this setup, but documents skeleton/load-order sensitivity.

---

### 3. Compiled behavior output — present at string level; graph unverified

**Files verified on disk** (2026-07-07 regen, `PreviousOutput.txt` + size
check):

| File | Size | Key ASCII counts |
|------|------|------------------|
| `0_Master.hkx` | 585,136 B | IdleFurnitureStart **1**, IdleFurnitureExit **1**, HorseEnter **7** |
| `mt_behavior.hkx` | 1,157,440 B | IdleFurnitureExit **3**, to_FurnitureState **1**, FurnitureState **2**, IdleFurnitureStart **0** |
| `horsebehavior.hkx` | 209,792 B | HorseEnter **6** |
| All three | — | Nemesis_Furniture **0** (expected — no FNIS `fu` mods in list) |

**Comparison baselines (same counts on mt_behavior / 0_Master markers):**

- Pandora PBE+ template HKX (`mods/Pandora Behaviour Engine+/Pandora_Engine/Skyrim/Template/...`)
- Nemesis SSE cache (`Project New Reign.../cached_behaviors/SSE/`)

**0_Master context:** `IdleFurnitureStart` sits in the expected idle/event
string neighborhood (between `IdleWebAlertOut` and `NextClip`) on Output,
template, and Nemesis cache.

**MO2 override scan:** `Pandora Output` is the **first enabled mod** in
`modlist.txt`. Only later behavior HKX is Serana hood FNIS plugin
(`FNIS_SeranaHoodFixWithAnim_Behavior.hkx`) — does not replace core graphs.

**Nemesis behavior templates:** `behavior templates/fu/` (with
`fu_master.txt` → `Nemesis_FurnitureState`, `#5195` MT_RootBehavior,
`#0089` wildcard transitions) exists **only** in the **disabled** MO2 mod
`Project New Reign - Nemesis Unlimited Behavior Engine`. **Not present** under
`tools/Pandora/` (only `Skyrim/Template` anim-data + precompiled HKX). With
no FNIS `fu` inputs on this list, absence of `Nemesis_Furniture*` strings in
output is **expected** even on a healthy Nemesis build — not diagnostic by
itself.

**Limitation:** String presence ≠ working state-machine transitions. Full
graph diff (HKX→XML) was **not** performed — this is the gap task-0031
must close.
