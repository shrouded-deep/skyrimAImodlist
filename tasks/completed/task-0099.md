## Problem

The donor graphics block (above `-Graphics Improvements_separator`) was inserted in
prefix-number order (016 first, 066 last). In MO2's modlist.txt the block ended up
with the **lowest-prefix group nearest the top** and the **highest-prefix group
nearest the anchor** — which is the reverse of what MO2 needs.

Current order reading top-to-bottom in modlist.txt (wrong):

```
... (rest of modlist above)
+[Visual Foundations mods — prefix 016]
-Visual Foundations_separator
...
+[Visuals - Late Loaders mods — prefix 066]
-Visuals - Late Loaders_separator
-Graphics Improvements_separator   ← anchor, stays fixed
```

Required order (corrected):

```
... (rest of modlist above)
+[Visuals - Late Loaders mods — prefix 066]
-Visuals - Late Loaders_separator
...
+[Visual Foundations mods — prefix 016]
-Visual Foundations_separator
-Graphics Improvements_separator   ← anchor, stays fixed
```

In short: **reverse the entire graphics block** so higher-prefix groups (later
loaders) sit above lower-prefix groups (base layers), and within each group the
mod lines are also reversed so their relative order matches the original donor
list.

**Profile:** `Lost Legacy - Fork` at `D:\Skyrim\`
**MO2 must be closed before editing modlist.txt.**

---

## What to change in modlist.txt

### Scope

Only the lines between (and not including) the last non-graphics mod above the
block and the `-Graphics Improvements_separator` line. The anchor and everything
outside the graphics block must not move.

### Algorithm

1. Collect every line from the start of the new donor graphics block down to
   (but not including) `-Graphics Improvements_separator`.
2. Split into sub-groups: each group is one separator line plus all the mod lines
   that sit above it (i.e. belong to it). A group boundary is a line matching
   `-*_separator`.
3. Reverse the list of groups.
4. Within each group, also reverse the order of the mod lines (the separator line
   itself stays at the bottom of its group).
5. Write the result back — groups in reversed order, mods within each group in
   reversed order, anchor unchanged.

### Invariants

- `-Graphics Improvements_separator` does not move.
- No mod lines are added or removed — same set, reordered only.
- Separator lines keep their `-` prefix (disabled).
- Mod enable/disable state (`+`/`-` prefix on each mod line) is preserved as-is.
- `plugins.txt` is **not** touched — plugin load order is independent.
