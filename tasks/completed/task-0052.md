### Three defects found and fixed

**1. `Vokriinator Black.esp` missing — CRITICAL (VB stack broken at its core).**
The `Vokriinator Black` mod folder existed but was **incomplete**: Cursor's install
script copied the 5 loose `.pex` scripts but **not `Vokriinator Black.esp` itself**
(529 KB merged plugin), meta.ini, or the source `.psc` scripts. The central VB merge
plugin — which every VB patch depends on and which provides the actual merged perk
trees — was absent. **Fix:** robocopy'd the complete folder from
`E:\Nolvus\Nolvus\MODS\mods\Vokriinator Black\`. esp now present, warning cleared.

**2. `Captial Whiterun and Spaghetti AIO Patch.esp` — missing master.**
Masters `Spaghetti's Cities - AIO.esp`, which is not active (Cursor correctly used the
modular Spaghetti ESPs). This patch is the wrong variant — built for Spaghetti AIO, not
the modular split. The installed `CWE Spaghetti Patch` folder contains **only** the AIO
patch; no modular variant, and no source archive in the shared cache. **Fix:** disabled
the `CWE Spaghetti Patch` mod + deactivated its plugin. Follow-up task-0053 to source
the correct modular CWE×Spaghetti patch (or confirm none needed).

**3. `Description Framework - Keizaal Cut.esp` — stale active entry.**
task-0047 disabled this mod (unparsable stub) but its plugin line stayed active in
Fork's plugins.txt because the F5 refresh ran on the Keizaal profile. **Fix:**
deactivated the plugin line in Fork plugins.txt + loadorder.txt.

### Systemic concern → task-0054

Defect #1 proves Cursor's install scripts can do a **partial folder copy** (scripts
copied, esp skipped). The resolver only checks *plugin* files — a folder that copied its
esp but not its meshes/textures/scripts would pass this audit yet cause missing assets
(purple textures, broken FX, dead scripts) in-game. **task-0054** queued to run a
folder-integrity check comparing all script-copied Keizaal mod folders against their
Nolvus sources (file count + total size per folder).
