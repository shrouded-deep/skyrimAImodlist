## Description

The baseline scan (task-0002) found that `modlist/load-order-notes.md`
documents "bashed/smashed patch last" as part of the tiering methodology,
but the actual installed load order has no such plugin — it ends with
Synthesis patchers, ParallaxGen, DynDOLOD, and an Occlusion patch instead.

This needs a deliberate decision, not a silent correction. Research
whether Synthesis-based builds conventionally replace the traditional
xEdit/Wrye Bash bashed/smashed patch step (Synthesis patchers can cover
overlapping ground — e.g. leveled list merging — that a bashed patch
traditionally handles), or whether this list is genuinely missing a step
it should have.

## Acceptance criteria

- [x] Research how Synthesis patchers relate to traditional bashed/
      smashed patches — do they supersede it, complement it, or leave a
      gap depending on which Synthesis patchers are actually installed?
- [x] Check which specific Synthesis patchers are present in this list
      (see `modlist/baseline-plugins-2026-07-05.txt`) and what each one
      covers
- [x] Write a recommendation: either (a) the current setup is correct and
      no bashed/smashed patch is needed, or (b) one should be added, with
      reasoning
- [x] Update `modlist/load-order-notes.md` to reflect the actual, verified
      methodology — don't leave the stale "bashed/smashed last" language
      if it's wrong
- [x] Add an entry to `modlist/decisions.md` documenting the decision and
      why

## Result

**Recommendation: (a) current setup is correct — no bashed/smashed patch
needed by default.**

Research summary:
- Synthesis and bashed/smashed patches **complement** each other; official
  Synthesis docs recommend bashed patch *before* Synthesis when both exist.
  Synthesis does not generically supersede leveled-list merging.
- Anvil ships **no Wrye Bash tool** and **no bashed/smashed plugin**. Post-
  install docs mention Synthesis but never instruct users to build a Bash
  Patch.
- Four pre-generated Synthesis outputs (`ANV_SynHPHRaceMenuPatcher.esp`,
  `ANV_SynNPCPatcher.esp`, `ANV_SynWorldPatcher.esp`, `ANV_SynW4ENBPatcher.esp`)
  contain specialized patchers only (RaceMenu/HPH, NPC fixes, SSBG/world,
  Water-for-ENB) — verified via `Anvil\tools\Synthesis\PipelineSettings.json`.
- Anvil handles merge-style concerns through named compatibility patches,
  pre-merged plugins (`SMIM-SE-Merged-All.esp`, etc.), and load-order
  curation. Baseline scan found no unintentional conflicts in audited categories.

**Files changed:**
- `modlist/load-order-notes.md` — replaced stale "bashed/smashed last"
  with verified tiering (framework first, preserve Anvil order, final tier
  ParallaxGen → Synthesis → DynDOLOD → Occlusion; Synthesis patcher table)
- `modlist/decisions.md` — new entry documenting the decision

**Follow-up (not created):** If heavy customization adds many leveled-list
or inventory mods, evaluate Fusion Patcher / LeveledListResolver (Synthesis)
or Wrye Bash as an explicit task — not a silent default.
