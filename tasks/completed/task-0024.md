## Description

During Lux Via/MLO2 cleanup, one of the two custom `ANV_Syn*.esp` patches
flagged with a missing Lux master was found in Synthesis to reference a
source file located on a path specific to someone else's local
filesystem (not present in this install, not part of any known
distributable content). The other flagged patch is confirmed safe to
re-run through Synthesis normally.

(See task file history for DynDOLOD missing-master notes — handled separately
via task-0025 deferred / profile disable.)

## Result

### Broken patch identified

| Item | Detail |
|---|---|
| Output plugin | `ANV_SynW4ENBPatcher.esp` (`Anvil - Synthesis Output/`, ~894 KB) |
| Synthesis group | `ANV_SynW4ENBPatcher` |
| Broken setting | `SolutionPatcherSettings.SolutionPath` → `F:\_Modding Tools\Skyrim Modding Tools\Patchers\WaterForENBPatcherFixed\WaterForENBPatcher.sln` |
| Nickname in config | `WaterForENBPatcherFixed` (local fork label) |

**What the patcher does** (official [panthuncia/WaterForENBPatcher](https://github.com/panthuncia/WaterForENBPatcher), documented on [Water for ENB Nexus 37061](https://www.nexusmods.com/skyrimspecialedition/mods/37061)): forwards **Water for ENB** water records — **CELL** `WaterEnvironmentMap`, **WRLD** `LodWater`, and water-flagged **CELL** refs under Tamriel/worldspace blocks — from the winning load-order mod into a generated patch so downstream mods do not strip W4ENB water data.

**Current baked output masters:** `Skyrim.esm`, DLCs, `Water for ENB (Shades of Skyrim).esp`, `VolkiharSoundscapeOverhaul.esp` — no Lux masters.

### Classification: **(b) portable path fix**

The patcher source is the **public GitHub repo** `panthuncia/WaterForENBPatcher`. Anvil's `WaterForENBPatcherFixed` name/path is a **maintainer-local clone** on drive `F:\` — not shipped with the list, not in Anvil's public repo docs. No separate public "Fixed" fork found; treat as same upstream project (possibly with uncommitted local edits — **regen diff recommended** at Tier 2 before trusting output).

### Anvil upstream check

[Oghma-Infinium/Anvil](https://github.com/Oghma-Infinium/Anvil) README documents Synthesis as an MO2 tool but **does not document** `PipelineSettings.json` patcher paths or the `F:\` dependency. **Known portability defect** in the Wabbajack list artifact, not documented upstream.

### Fix applied (config only — no regen)

Updated `Anvil\tools\Synthesis\PipelineSettings.json`:

- Replaced `SolutionPatcherSettings` (`F:\...Fixed...`) with **`GithubPatcherSettings`**
- Repo: `https://github.com/panthuncia/WaterForENBPatcher`
- Project: `WaterForENBPatcher\WaterForENBPatcher.csproj`
- Pinned commit: `111d09c88a168bf37c79b199afd97ab80b1a251c` (main, WENB 2.18 support)

**`ANV_SynW4ENBPatcher.esp` left unchanged** — frozen static output until Tier 2 Synthesis window. Regenerating with upstream GitHub may differ from legacy "Fixed" fork; compare before replacing.
