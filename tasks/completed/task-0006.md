## Description

Before the human runs Tier 2 regen (Synthesis, DynDOLOD, TexGen,
ParallaxGen), Successor-profile output folders must exist so the tools
write into Successor copies and never overwrite the pristine Nolvus
Awakening originals.

The current disabled Nolvus output folders are:

| Nolvus folder (disabled, do not touch) | New Successor folder to create |
|---|---|
| `DynDOLOD - Output - Ultimate` | `Successor - DynDOLOD Output` |
| `DynDOLOD - Textures - Ultimate` | `Successor - DynDOLOD Textures` |
| `Synthesis Patch` | `Successor - Synthesis Patch` |

A ParallaxGen output folder will also be needed. The Nolvus original is:

| Nolvus folder | New Successor folder |
|---|---|
| `ParallaxGen Output` (or similar — check modlist.txt for the actual name) | `Successor - ParallaxGen Output` |

MO2 paths:
- Mods root: `E:\Nolvus\Nolvus\MODS\mods\`
- Successor modlist: `E:\Nolvus\Nolvus\MODS\profiles\Successor\modlist.txt`

## Steps

1. **Assert MO2 is closed** before touching any files.
2. For each output folder pair above:
   a. Create a new **empty** folder under `E:\Nolvus\Nolvus\MODS\mods\`
      with the `Successor -` name. Do **not** copy any files from the
      Nolvus original — the regen tools will populate them fresh.
   b. Locate the Nolvus original folder's position in Successor
      `modlist.txt` (it will be present as a `-` disabled line).
   c. Insert the new Successor folder as a **disabled** (`-`) entry
      **directly below** the Nolvus original's line, so it sits in the
      same position group ready to be enabled after regen.
3. Verify ParallaxGen folder name by checking `modlist.txt` for a line
   containing "ParallaxGen" — create the Successor equivalent only if
   the Nolvus original is present.
4. Do **not** enable any of the new folders in modlist.txt — leave them
   disabled (`-`). The human will enable them after regen completes.
5. Do **not** modify `plugins.txt` — the new output folders have no
   plugins yet.
