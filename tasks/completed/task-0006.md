## Result

conflicts (Armor 662, Weapon 607, ConstructibleObject 36, LeveledItem 137)
using the same winner-grouping + spot-check method as task-0004/task-0005.

**1,438 of 1,442 confirmed healthy.** USSEP (1,114 combined across all
four types), USMP (25 combined), DLC/master overrides (Update/Dawnguard/
HearthFires/Dragonborn — 121 combined), and named mods paired with their
own USSEP patches: `JS Unique Utopia SE - Rings/Daggers - Johnskyrim.esp`,
`JS Vanilla Circlets SE.esp`, `Praedy's StavesAIO.esp`,
`ForswornRetexture Unique Armor of the Old Gods.esp`,
`EnchantableSpecialItemFix_USSEP.esp`, `Hearthfires Houses Building
Fix.esp`, `SurvivalModeImproved.esp`. Spot-checked the less obviously-named
ones (`BeardMaskFix.esp`, `PraedysSkeletons.esp`,
`ArcaneBlacksmithHoodFix.esp`) via `conflict_tree` and confirmed each wins
only records exactly matching its name/purpose.

**4 flagged — needs verification (low severity):** `Lux Orbis.esp` wins 4
LeveledItem conflicts unrelated to lighting (`GuardGear`,
`CWSoldierImperialGear`, `CWSoldierSonsGear`, `CWSoldierSonsGear1H`),
each removing an ammo entry (confirmed for `GuardGear`: drops a 12x Steel
Arrow entry vs. vanilla). Checked Lux Orbis via `housecarl_nexus_search`
(Nexus id 56095) — it's a large, actively-maintained lighting overhaul
whose FOMOD bundles patches for a lot of modded content, so this is
plausibly an intended FOMOD option, but not traceable to an obviously
dedicated patch from the load order alone. Not treated as a confirmed bug
(low severity, unlike task-0005's McmRecorder finding) — filed as
task-0010 for Cursor to confirm via documentation rather than assuming.
